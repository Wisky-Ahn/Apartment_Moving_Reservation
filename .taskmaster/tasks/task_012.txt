# Task ID: 12
# Title: 공지사항 페이지 기능 및 UI 문제 해결
# Status: done
# Dependencies: 9, 11
# Priority: high
# Description: 공지사항 페이지의 중복된 header 제거, 수정/삭제/추가 기능 복구, 공지 내용 보기 및 "더 찾아보기" 버튼 상호작용 문제를 해결하여 완전한 기능을 구현합니다.
# Details:
1. 중복 header 제거:
   - 공지사항 페이지에서 navbar와 중복되는 header 요소 식별
   - 중복 header 컴포넌트 제거 및 레이아웃 조정
   - 페이지 구조 재조정 후 스타일 일관성 확인

2. 공지사항 CRUD 기능 복구:
   - 수정 기능:
     - 공지사항 수정 폼 컴포넌트 점검
     - 수정 API 엔드포인트 연결 확인 및 수정
     - 권한 검증 로직 구현 (관리자만 수정 가능)
     - 수정 후 상태 업데이트 및 UI 반영 로직 구현
   - 삭제 기능:
     - 삭제 확인 모달 구현
     - 삭제 API 엔드포인트 연결 확인 및 수정
     - 삭제 후 목록 업데이트 로직 구현
   - 게시글 추가 기능:
     - 공지사항 작성 폼 컴포넌트 점검
     - 필수 입력 필드 검증 로직 추가
     - 작성 API 엔드포인트 연결 확인 및 수정
     - 작성 완료 후 목록 업데이트 및 리디렉션 구현

3. 공지 내용 보기 기능 복구:
   - 행 선택 이벤트 핸들러 점검 및 수정
   - 공지사항 상세 내용 표시 컴포넌트 구현
   - 선택된 공지사항 데이터 가져오는 로직 수정
   - 모바일/데스크톱 환경에서 상세 보기 UI 최적화

4. "공지사항 더 찾아보기" 버튼 기능 구현:
   - 버튼 이벤트 핸들러 추가
   - 페이지네이션 또는 무한 스크롤 기능 구현
   - 추가 공지사항 로드 API 연결
   - 로딩 상태 표시 및 에러 처리

5. 전체 공지사항 페이지 UI/UX 개선:
   - 반응형 디자인 점검 및 수정
   - 접근성(a11y) 개선 (키보드 탐색, 스크린 리더 지원)
   - 로딩 상태 및 에러 메시지 UI 개선
   - 사용자 피드백 요소 추가 (토스트 메시지 등)

# Test Strategy:
1. 중복 header 제거 테스트:
   - 공지사항 페이지 로드 시 header가 한 번만 표시되는지 확인
   - 다양한 화면 크기에서 레이아웃이 올바르게 표시되는지 확인
   - 스타일 일관성 검증

2. CRUD 기능 테스트:
   - 수정 기능:
     - 관리자 계정으로 로그인하여 공지사항 수정 버튼 클릭
     - 제목, 내용 등 필드 수정 후 저장
     - 변경사항이 즉시 반영되는지 확인
     - 비관리자 계정으로 수정 버튼이 보이지 않는지 확인
   - 삭제 기능:
     - 관리자 계정으로 공지사항 삭제 버튼 클릭
     - 확인 모달에서 삭제 확인
     - 목록에서 해당 항목이 제거되는지 확인
   - 추가 기능:
     - 새 공지사항 작성 폼 열기
     - 필수 필드 누락 시 적절한 오류 메시지 표시 확인
     - 모든 필드 입력 후 저장 시 목록에 새 항목 추가 확인

3. 공지 내용 보기 테스트:
   - 공지사항 목록에서 항목 클릭
   - 상세 내용이 올바르게 표시되는지 확인
   - 다른 항목 클릭 시 내용이 변경되는지 확인
   - 모바일 기기에서 상세 보기 레이아웃 확인

4. "더 찾아보기" 버튼 테스트:
   - 버튼 클릭 시 추가 공지사항이 로드되는지 확인
   - 로딩 상태가 적절히 표시되는지 확인
   - 모든 공지사항을 로드한 후 버튼 상태/표시 변화 확인
   - 네트워크 오류 시 적절한 오류 메시지 표시 확인

5. 통합 테스트:
   - 다양한 브라우저(Chrome, Firefox, Safari, Edge)에서 모든 기능 테스트
   - 모바일 및 태블릿 환경에서 반응형 디자인 테스트
   - 키보드만으로 모든 기능 사용 가능한지 접근성 테스트
   - 페이지 로드 성능 및 상호작용 지연시간 측정
